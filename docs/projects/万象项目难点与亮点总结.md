## 难点一：可拖拽数据看板编辑器 - 多图表类型与实时预览

### Situation（背景）
业务方需要一个灵活的数据看板系统，支持拖拽式布局，能够展示 8 种不同类型的图表（柱状图、饼图、折线图、表格、甘特图、散点图、指标卡、卡片），并且需要实时预览数据。用户可以从左侧指标库拖拽指标到画布，自动生成对应的图表。

### 难点拆解

#### 难点 1.1：左侧指标库动态加载
**问题**：
- 不同图表类型对应不同的指标库（如柱状图有 20+ 个指标，表格有 30+ 个指标）
- 切换图表类型时需要重新加载指标库
- 指标库包含缩略图，需要优化加载性能

**拆解思路**：
1. 按图表类型分组加载指标
2. 使用图片懒加载优化缩略图加载
3. 缓存已加载的指标数据

#### 难点 1.2：拖拽到画布的碰撞检测
**问题**：
- 从左侧拖拽指标到画布时，需要自动计算位置
- 避免与现有图表重叠
- 拖拽过程中需要显示占位符

**拆解思路**：
1. 使用 vue-grid-layout 的碰撞检测算法
2. 手动实现简化版碰撞检测作为补充
3. 拖拽时显示虚拟占位符

#### 难点 1.3：图表宽度调整后的自动排列
**问题**：
- 用户调整图表宽度后，下方图表需要自动上移填补空白
- 需要避免图表重叠
- 调整后需要触发图表重绘

**拆解思路**：
1. 实现自定义的 compact 算法
2. 按 Y 坐标排序后逐个计算位置
3. 调整完成后触发 resize 事件

#### 方案 2：拖拽碰撞检测与自动排列

**拖拽事件处理**：
```javascript
drag(e, item, i) {
  let cate = this.echartTypeList[this.activeDragIndex]
  
  // 1. 获取画布位置
  let parentRect = document.getElementById('content').getBoundingClientRect()
  let mouseInGrid = false
  
  // 2. 判断鼠标是否在画布内
  if (mouseXY.x > parentRect.left && mouseXY.x < parentRect.right &&
      mouseXY.y > parentRect.top && mouseXY.y < parentRect.bottom) {
    mouseInGrid = true
  }
  
  // 3. 在画布内且未创建占位符时，创建占位符
  if (mouseInGrid === true && this.layout.findIndex((item) => item.i === 'drop') === -1) {
    this.layout.push({
      x: (this.layout.length * 2) % 12,
      y: this.layout.length + 12,
      w: cate.w,
      h: cate.h,
      i: 'drop'  // 占位符标识
    })
  }
  
  // 4. 更新占位符位置
  let index = this.layout.findIndex((item) => item.i === 'drop')
  if (index !== -1) {
    // 隐藏占位符的 DOM
    this.$refs.gridlayout.$children[this.layout.length].$refs.item.style.display = 'none'
    
    // 获取占位符实例
    let el = this.$refs.gridlayout.$children[index]
    el.dragging = { 
      top: mouseXY.y - parentRect.top, 
      left: mouseXY.x - parentRect.left 
    }
    
    // 转换为网格坐标
    let new_pos = el.calcXY(mouseXY.y - parentRect.top, mouseXY.x - parentRect.left)
    
    // 触发布局引擎计算
    this.$refs.gridlayout.dragEvent('dragstart', 'drop', new_pos.x, new_pos.y, cate.h, cate.w)
    DragPos.i = String(index)
    DragPos.x = this.layout[index].x
    DragPos.y = this.layout[index].y
  }
}

dragend(e, item, i) {
  // 1. 移除占位符
  let index = this.layout.findIndex((item) => item.i === 'drop')
  if (index !== -1) {
    this.layout.splice(index, 1)
  }
  
  // 2. 创建真实图表
  if (DragPos.i !== null) {
    const newItem = {
      x: DragPos.x,
      y: DragPos.y,
      w: item.w || 6,
      h: item.h || 21,
      i: this.layout.length + 1,
      chartType: this.echartTypeList[this.activeDragIndex].name,
      chartName: item.metric,
      metricId: item.metricId,
      loading: false,
      chartData: '',
      // ... 其他配置
    }
    
    this.layout.push(newItem)
    
    // 3. 选中新创建的图表
    this.activeIndex = newItem.i
    this.currentComponent = newItem
  }
  
  DragPos.i = null
}
```

**自动排列算法**（手动实现的 compact）：
```javascript
updateLayout(newItem) {
  // 1. 边界检查：确保不超出画布
  if (12 - newItem.x < newItem.w) {
    newItem.x = 12 - newItem.w
  }
  
  // 2. 标记为静态，避免被移动
  newItem.static = true
  
  // 3. 获取所有静态元素（已固定位置的元素）
  const compareWith = this.getStatics(this.layout)
  
  // 4. 按 Y 坐标和 X 坐标排序
  const sorted = this.sortLayoutItemsByRowCol(this.layout)
  const out = Array(this.layout.length)
  
  // 5. 逐个处理元素，检测碰撞并调整位置
  for (let i = 0, len = sorted.length; i < len; i++) {
    let l = sorted[i]
    
    if (!l.static) {
      // 非静态元素，执行碰撞检测
      l = this.compactItem(compareWith, l)
      compareWith.push(l)
    }
    
    out[this.layout.indexOf(l)] = l
    l.moved = false
  }
  
  // 6. 恢复为非静态
  newItem.static = false
  
  // 7. 触发图表重绘
  this.$nextTick(() => {
    this.$refs['chartBox' + newItem.i][0].resize()
  })
}

// 碰撞检测：将元素向下移动直到不碰撞
compactItem(compareWith, l) {
  let collides
  while ((collides = this.getFirstCollision(compareWith, l))) {
    l.y = collides.y + collides.h  // 移动到碰撞元素下方
  }
  return l
}

// 检测两个元素是否碰撞
collides(l1, l2) {
  if (l1 === l2) return false
  if (l1.x + l1.w <= l2.x) return false  // l1 在 l2 左侧
  if (l1.x >= l2.x + l2.w) return false  // l1 在 l2 右侧
  if (l1.y + l1.h <= l2.y) return false  // l1 在 l2 上方
  if (l1.y >= l2.y + l2.h) return false  // l1 在 l2 下方
  return true  // 有重叠
}
```

**关键点**：
- 使用占位符 `i: 'drop'` 标识拖拽中的元素
- 手动实现碰撞检测算法，补充 vue-grid-layout 的不足
- 调整完成后触发 `resize()` 重绘图表

## 难点二：甘特图集成 - 大数据量渲染与交互优化

### Situation（背景）
业务方需要在看板中集成甘特图，展示项目进度和里程碑。甘特图需要支持 1000+ 条数据的渲染，并且需要支持拖拽调整任务时间、展开/收起子任务等交互。使用 dhtmlxGantt 库后，发现大数据量渲染时性能极差，页面卡顿严重。

### 难点拆解

#### 难点 2.1：大数据量渲染性能问题
**问题**：
- 1000 条数据渲染时间 >10s
- 页面卡顿，无法响应用户操作
- 内存占用过高，容易崩溃

**拆解思路**：
1. 限制渲染数量（前 1000 条）
2. 使用虚拟滚动
3. 延迟渲染非可见区域

#### 难点 2.2：甘特图与 Vue 的集成
**问题**：
- dhtmlxGantt 是原生 JS 库，与 Vue 的响应式系统冲突
- 数据更新时需要手动触发重绘
- 组件销毁时需要手动清理资源

**拆解思路**：
1. 封装为 Vue 组件
2. 使用 `$watch` 监听数据变化
3. 在 `beforeDestroy` 中清理资源

#### 难点 2.3：甘特图的自适应布局
**问题**：
- 看板中的甘特图需要自适应容器大小
- 窗口 resize 时需要重新计算布局
- 切换看板时需要重新渲染

**拆解思路**：
1. 监听容器尺寸变化
2. 使用 `ResizeObserver` API
3. 防抖优化 resize 事件

<script>

import 'dhtmlx-gantt/codebase/dhtmlxgantt.css'

**防抖优化**：
```javascript
import { debounce } from 'lodash'

export default {
  mounted() {
    // 监听窗口 resize
    this.handleResize = debounce(() => {
      this.layout.forEach((item, index) => {
        this.$refs['chartBox' + item.i][0].resize()
      })
    }, 300)
    
    window.addEventListener('resize', this.handleResize)
  },
  
  beforeDestroy() {
    window.removeEventListener('resize', this.handleResize)
  }
}
```

**关键点**：
- 使用防抖优化 resize 事件，避免频繁触发

- 统一的 resize 接口，便于维护

## 难点三：富文本编辑器集成 - TinyMCE 深度定制

### Situation（背景）
项目需要集成富文本编辑器，用于工作项描述、反馈内容、日程备注等场景。使用 TinyMCE 后，发现默认配置无法满足需求：需要支持图片上传、@提及用户、表格编辑、代码高亮等功能，并且需要与 Vue 的响应式系统良好集成。

### 难点拆解

#### 难点 3.1：图片上传与粘贴
**问题**：
- 需要支持图片上传到服务器
- 需要支持粘贴图片
- 需要支持拖拽图片

**拆解思路**：
1. 配置 `images_upload_handler` 自定义上传逻辑
2. 配置 `paste_data_images: true` 支持粘贴图片
3. 使用 FormData 上传图片

#### 难点 3.2：编辑器高度自适应
**问题**：
- 不同场景需要不同的编辑器高度
- 内容增多时需要自动扩展高度
- 需要支持最小高度和最大高度

**拆解思路**：
1. 使用 `autoresize` 插件
2. 通过 props 传递高度参数
3. 配置 `autoresize_bottom_margin` 控制底部边距

#### 难点 3.3：只读模式与编辑模式切换
**问题**：
- 查看工作项时需要只读模式
- 编辑工作项时需要编辑模式
- 切换时需要保留内容

**拆解思路**：
1. 通过 props 传递 `readonly` 参数
2. 使用 `setup` 钩子动态设置只读状态
3. 使用 `v-model` 双向绑定内容

  methods: {
    // 获取文件图标
    getFileIcon(fileName) {
      const ext = fileName.split('.').pop().toLowerCase()
      const iconMap = {
        pdf: 'icon-pdf',
        doc: 'icon-word',
        docx: 'icon-word',
        xls: 'icon-excel',
        xlsx: 'icon-excel',
        ppt: 'icon-ppt',
        pptx: 'icon-ppt',
        zip: 'icon-zip',
        rar: 'icon-zip',
        jpg: 'icon-image',
        jpeg: 'icon-image',
        png: 'icon-image',
        gif: 'icon-image'
      }
      return iconMap[ext] || 'icon-file'
    },
    
    // 格式化文件大小
    formatFileSize(bytes) {
      if (bytes === 0) return '0 B'
      const k = 1024
      const sizes = ['B', 'KB', 'MB', 'GB']
      const i = Math.floor(Math.log(bytes) / Math.log(k))
      return (bytes / Math.pow(k, i)).toFixed(2) + ' ' + sizes[i]
    },
    
    // 文件类型校验
    validateFileType(file, allowedTypes) {
      const ext = file.name.split('.').pop().toLowerCase()
      return allowedTypes.includes(ext)
    },
    
    // 文件大小校验
    validateFileSize(file, maxSize) {
      return file.size <= maxSize * 1024 * 1024
    },
    
    // 下载文件
    downloadFile(url, fileName) {
      const link = document.createElement('a')
      link.href = url
      link.download = fileName
      link.click()
    }
  }
