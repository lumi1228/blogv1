Koa介绍&核心概念

Koa 是一个新的 web 框架，致力于成为 web 应用和 API 开发领域中的一个更小、更富有表现力、更健壮的基石。
利用 async 函数丢弃回调函数，并增强错误处理。 Koa 没有任何预置的中间件，可快速而愉快地编写服务端应用程序。

特点：

  • 简洁
  • 支持async/await语法
  • 丰富的中间件

主要概念：

  • Koa Application（应用程序）
  • Context（上下文）
  • Request（请求）、Response（响应）

安装与使用

  • 初始化项目&安装koa，生成 package.json
$ cd koa-demo
$ npm init -y
$ npm install --save koa // 安装koa到生产环境
  • 根目录创建index.js，编写Koa应用
@/index.js
const Koa = require('koa')
const app = new Koa()

app.use(async ctx=>{
    ctx.body = 'hello koa!'
})

app.listen(3000)
  • 启动应用$ node index.js后，访问http://localhost:3000/,网页中显示hello koa!

Koa工作原理

  • 验证中间件执行顺序：
@/middleware.js
const Koa = require('koa')
const app = new Koa()

const middleware = async (ctx, next) => {
    console.log('this is middleware')
    next() // 调用下一个中间件
    console.log('this is middleware after next')
}
const middleware1 = async (ctx, next) => {
    console.log('this is middleware1')
    next()
    console.log('this is middleware1 after next')
}

const middleware2 = async (ctx, next) => {
    console.log('this is middleware2')
    next()
    console.log('this is middleware2 after next')
}


app.use(middleware)
app.use(middleware1)
app.use(middleware2)

app.listen(3000)

// 运行结果：
this is middleware
this is middleware1
this is middleware2
this is middleware2 after next
this is middleware1 after next
this is middleware after next
  • 运行node middleware，结果分析：
    • 按照app.use()注册的顺序执行
    • next()调用后才执行下一个中间件，若无next()，则中间件执行完后不再继续
    • next()后的代码，在next()调用后才执行，按照顺序先进后出

使用中间件koa-router、koa-body、@koa/cors、koa-json

本节是借助koa中间件实现一个简易的开发RESTful接口

  • 路由：koa-router
  • 协议解析：koa-body
  • 跨域处理：@koa/cors
  • JSON处理：koa-json
  • 安装中间件
$ npm install --S koa-router koa-body @koa/cors koa-json  // --S等价于--save
  • 编写路由规则
@/index.js
const Koa = require('koa')
const Router = require('koa-router')
const { koaBody } = require('koa-body')
const cors = require('@koa/cors')
const json = require('koa-json')

const app = new Koa()
const router = new Router()
// const bodyParser = new body()

router.prefix('/api') // 定义路由前缀

router.get('/', async ctx=>{ // 定义路由
    console.log(ctx)
    // console.log(ctx.request)
    ctx.body = 'hello koa!'
})

router.get('/api', async ctx=>{ // 定义路由
    //get params
    const params = ctx.request.query
    console.log(params)
    ctx.body = {
        ...params,
        message: 'hello api!'
    }
})

router.post('/post', async ctx=>{
    const { body } = ctx.request // 解析请求体，拿到客户端提交的数据
    console.log(body) 
    console.log(ctx.request)
    ctx.body = { // 响应体设置，把处理结果作为响应返回
        ...body,
        message: 'hello post!'
    }
}) 


// app.use 会按照注册的顺序依次执行
app.use(koaBody())// 解析请求体
app.use(cors())// 处理跨域请求
app.use(json({pretty: false, param: 'pretty'}))// 处理JSON响应，pretty: false 不格式化JSON，param: 'pretty' 参数名
app.use(router.routes()) 
    .use(router.allowedMethods()) // 使用路由允许的方法

app.listen(3000)
  • 启动应用后，访问http://localhost:3000/,网页中显示hello koa!;
  • 访问http://localhost:3000/api/api,网页中显示hello api!;
  • postman访问http://localhost:3000/api/post,返回请求体的body数据
